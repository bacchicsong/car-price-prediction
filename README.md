# Проект по предсказанию цен на автомобили

## Описание проекта

Данный проект включает в себя решение задачи по предсказанию цен автомобилей.
Состоит из следующих пунктов:
- Exploratory Data Analysis (EDA)
- Предобработка данных
- Обучение и сравнение моделей
- Развёртывание Streamlit-приложения

## Что было сделано

### 1. EDA и предобработка данных
- Анализ пропущенных значений (заполнение медианами из train)
- Удаление дубликатов по признаковому описанию
- Преобразование типов данных
- Удаление столбца `torque`
- Анализ корреляций и построение визуализаций

### 2. Обучение моделей

Модель	Бизнес-Метрика
- Ridge (с категориями)	0.269
- ElasticNet (GridSearch)	0.254
- Linear Regression (скелинг)	0.235
- Linear Regression (без скелинга)	0.235
- Lasso (GridSearch)	0.235
- Lasso	0.235

**Описание метрики:**
Среди всех предсказанных цен на авто нужно посчитать долю прогнозов, отличающихся от реальных цен на эти авто не более чем на 10% (в одну или другую сторону)


### 3. Развёртывание
- Создано Streamlit-приложение с интерактивным интерфейсом
- Модель сохранена в pickle-формате для быстрого инференса

## Основные результаты

### Что дало наибольший прирост качества:
**Добавление категориальных признаков** 

### Ключевые инсайты из EDA:
- Автомобили с автоматической трансмиссией значительно дороже
- Сильная положительная корреляция между `max_power` и ценой
- Год выпуска - один из важнейших предикторов цены

## Что не удалось и почему

1. **Обработка столбца `torque`** - сложная структура
2. **Обработка столбца `name`** - слишком много уникальных значений, требуется группировка по брендам
3. **Нелинейные модели** - не рассматривались, но могли бы улучшить результат

## Оценка Streamlit-приложения

### Что получилось хорошо:
- Интуитивно понятный интерфейс ввода данных
- Визуализация важности признаков
- Отображение ключевых графиков EDA
- Возможность загрузки CSV файла

### Ограничения и проблемы:
- Нет валидации на некорректные значения признаков
- Отсутствует обработка выбросов при инференсе
- Предсказания могут быть отрицательными
  
### Планы на улучшение:
- Добавить обработку столбца `name` (извлечение бренда)
- Добавить доверительные интервалы для предсказаний
- Добавить анализ выбросов
- Рассмотреть нелинейные модели
